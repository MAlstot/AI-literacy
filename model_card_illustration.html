What Is a Model Card?

A model card is a brief document published alongside an AI model that discloses how it was built, what it's intended for, where it may fall short, and how it performs across different demographic groups. The example below illustrates the key sections of a model card — using a fictional model called EquityScore-3.

🤖

EquityScore-3 / hiring-assessment-v3

Released: March 2025 Version: 3.1.2 Developer: FictionalAI Research Lab

Model Summary What it is

EquityScore-3 is a text classification model designed to assist human reviewers in screening job applications by flagging relevant qualifications for specific role types. It is not a standalone decision-making tool and is intended for use only as a first-pass filter under active human review.

This model was developed as part of a research initiative examining fairness-aware hiring tools and has not been validated for deployment in regulated hiring contexts without additional testing.

Uses What it's for — and not for
✓ Intended Use
  • Assisting human reviewers with large applicant pools
  • Research on algorithmic fairness in hiring
  • Pilot programs with active oversight and auditing
✗ Out-of-Scope Use
  • Automated hiring decisions without human review
  • Screening for roles requiring specialized credentials
  • Any deployment without bias auditing
  • Use in legally protected employment contexts (EEOC)
Training Data Where it learned from

The model was trained on approximately 2.3 million anonymized job application records collected between 2015–2023 from three hiring platforms (details under NDA). Data was filtered to remove direct demographic identifiers.

📄 Resume corpus (1.8M)

✓ Outcome labels (human-verified)

⚠ Note: Historical hiring decisions used as outcome labels may reflect prior organizational biases. See Bias Evaluation below.

Bias Evaluation How it performs across groups

Performance was evaluated across demographic subgroups using applicant self-identification data from a held-out evaluation set (n=42,000). Demographic parity and false positive rates were measured.

Known Limitations What it doesn't do well
!
The model shows higher false negative rates for applicants whose resumes include non-standard formatting, career gaps, or language patterns associated with non-dominant English dialects.
!
Training data over-represents candidates from US and UK hiring contexts. Performance on applications from other regions has not been validated and may be unreliable.
!
Outcome labels were derived from historical hiring decisions, which may encode prior organizational biases. Debiasing techniques were applied but cannot fully eliminate this risk.
!
Model has not been evaluated for intersectional fairness (e.g., women of color, disabled applicants). Intersectional evaluation is recommended before any deployment.
Recommendations for Deployers

Organizations wishing to pilot this model should conduct an independent bias audit against their own applicant pool prior to deployment. Human review of all model-flagged decisions is required. The authors recommend quarterly performance monitoring disaggregated by available demographic subgroup data.
